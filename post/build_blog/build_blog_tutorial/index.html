GithubPages+Hugo搭建博客

Oct 17, 2017 · 2 min read · blog hugo  ·
Share on:

【2023-09-22更新】

如果你不会Web前端开发，也不愿意花钱买服务器，懒洋洋地做个伸手党，只希望专注于内容创作...... 那么Github Pages + Hugo绝对适合你！

Why Github Pages?

免费！创建和管理都很方便！

只需在Github上创建一个repository，然后将Hugo生成的静态页面相关文件push到项目上，一切就搞定啦！

注：个人主页username.github.io的话push到默认的master分支上，普通repository则push到该项目的gh-pages分支上并保证根目录有一个index.html文件。

Why Hugo?

目前流行的几个静态网站生成器（SSG——Static Site Generator）：Hugo、Hexo、Jekyll。

Hugo官方主页：https://gohugo.io

  • Hugo是使用Go语言编写的静态网站生成器。
  • 相对于Jekyll复杂的安装设置来说，Hugo进行需要下载一个二进制文件（hogo.exe）即可轻松调试和生成静态页面。
  • Hugo生成静态页码的效率很高，速度上来说Hugo>Hexo>Jekyll。
  • Hugo自带watch的调试模式，可以在修改Markdown的文章内容的同时，自动更新浏览器页面呈现最终效果。
  • 缺点是主题模板相对其他两个来说数量比较少，用的人感觉还不算特别多，中文社区支持也不是很多。

Prepared

（一）Git & Github

  1. 下载安装
  • Git下载：https://git-scm.com/download
  • TortoiseGit下载：https://tortoisegit.org/download/

安装完成后，在开始菜单里找到Git -> Git Bash，蹦出一个命令行窗口说明Git安装成功。

1#Git Bash命令行输入
2git config --global user.name "Your Name"
3git config --global user.email email@example.com

因为Git是分布式版本控制系统，所以，每个机器都必须自报家门：你的名字和Email地址。

注意git config命令的--global参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置。

直接在你的目标文件夹里右键Git Bash Here，即可在当前目录中创建命令行窗口。

  1. 创建Github账户

进入GitHub官方主页注册自己的GitHub账户，登陆个人主页http://github.com/username。

右上角点击+号新建仓库（New Repository），仓库命名为username.github.io，注意username需要与注册时保持一致。

  1. SSH Key
1#生成密钥
2ssh-keygen -t rsa -C "username@xxx.com" -f ~/.ssh/filename

并将public key的内容并添加到仓库的设置中。

（二）Hugo

详细安装指南请参考：https://gohugo.io/getting-started/installing/

Hugo有两种版本：standard and extended。如果所用的网站模板用到了Transpile Sass to CSS的功能，则需安装extended、并额外安装Dart Sass。

我用的Windows系统，安装Hugo有两种常用方法：

  1. 在Hugo Releases上下载最新版的二进制程序：https://github.com/gohugoio/hugo/releases/latest。根据是否带enxtended字样、操作系统等选择所需下载的压缩包。下载后解压，然后把解压目录添加到环境变量里。
  2. 使用Scoop包管理器来管理Windows软件，一行代码搞定安装和卸载，干净又迅速。打开Windows PowerShell命令行，注意不要手欠开管理员模式。
 1set-executionpolicy remotesigned -scope currentuser
 2# 这里可以替换到你自定义的安装目录
 3$env:SCOOP='D:\Scoop'
 4[environment]::setEnvironmentVariable('SCOOP',$env:SCOOP,'User')
 5iex (new-object net.webclient).downloadstring('https://get.scoop.sh')
 6# 可能需要开vpn来访问github
 7
 8# 一行代码即可安装和卸载
 9# 安装Hugo的standard版本，D:\Scoop\apps会出现hugo文件夹
10scoop install hugo
11#如需安装extended版本
12scoop install hugo-extended
13# 安装Dart Sass
14scoop install sass
15
16# 卸载Hugo超简单
17scoop uninstall hugo
18# 卸载Scoop超简单
19scoop uninstall scoop
20
21# 其他软件安装参考 https://github.com/h404bi/dorado/blob/master/README_CN.md

验证是否安装成功：Win+R输入cmd回车，打开命令行窗口/或者直接在目录里右键打开Git Bash

1hugo help

如果正常运行则安装成功

Quick Start

迅速建立一个简单的个人Blog。

  1. 初始化

1）从刚才新建的Github仓库中拿到地址，比如git@github.com:username/username.github.io.git，用git clone命令拷贝仓库到本地某个目录。

2）另外再创建一个目录，用于存放Hugo工程，然后在该目录下右键点击Git Bash Here，输入命令。

1# 新建网站
2hugo new site hugo_project
3# 切换目录
4cd hugo_project
5# 初始化仓库
6git init
  1. 下载网站模板

可以去Hugo Themes挑选一个赏心悦目的模板，在网页上查看它的demo展示，满意的话就找到类似下面的安装命令来下载并拷贝样例网站相关文件：

1# ananke主题模板
2git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
3cp -r themes/ananke/exampleSite/. ./
4
5# clarity主题模板
6git submodule add https://github.com/chipzoller/hugo-clarity themes/hugo-clarity
7cp -a themes/hugo-clarity/exampleSite/* . && rm -f config.toml
8
9# 备忘一个以前用过的模板 https://github.com/flysnow-org/maupassant-hugo.git
  1. 修改配置文件

打开hugo_project根目录下的config.toml文件，修改一些参数：

  • title：会显示在网页标签上
  • baseURL：需要修改为自己的github.io地址，username.github.io
  • theme：为使用的主题，对应主题目录名

其他自定义配置可以自己慢慢研究config.toml文件以及themes/xxx/layouts文件夹里的内容，进行相应修改。

  1. 写博客文章

用hugo命令新建markdown文章，会自动存放到content/post目录下。

当然也可直接手动在该目录下创建md文件并编辑

1hugo new post/my-first-post.md
  1. 生成网站

修改博客文章时，可以先生成本地的网站用于预览，请访问http://localhost:1313/。

它会实时同步markdown文件的保存更新，很直观方便。

1hugo server --watch

预览结果一切满意了，进行静态网页的最终生成，hugo_project里会出现一个public目录。

1hugo

注意：生成过程中可能遇到各种保存，根据log信息问问chatgpt好了。

  1. 网站提交

将public中全部内容拷贝到git clone的本地仓库里，然后commit、push。

Success!!

现在就可以打开你的个人主页网址username.github.io欣赏成果啦！
